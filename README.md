# Task Management App

A Flutter task management app with Firebase Authentication and Provider-based
state management.

## Features

- **Email/password authentication** — sign up, log in, log out, password reset
- **Real-time task list** — add, complete, and delete tasks with instant UI updates
- **Per-user task storage** — each account's tasks are kept separate on-device
- **Animated UI** — splash screen, task list transitions, and interactive feedback
- **Filter tasks** — All / Pending / Done

## Tech Stack

| Layer | Tool |
|---|---|
| State management | [Provider](https://pub.dev/packages/provider) |
| Authentication | [Firebase Auth](https://firebase.google.com/docs/auth) |
| Local storage | [shared_preferences](https://pub.dev/packages/shared_preferences) |

## Setup

### 1. Configure Firebase

`lib/firebase_options.dart` is a placeholder — the app won't run until it's
replaced with your own project config.

1. Create a project at [console.firebase.google.com](https://console.firebase.google.com).
2. Go to **Build → Authentication → Sign-in method** and enable **Email/Password**.
3. Install the FlutterFire CLI (one-time):
   ```bash
   dart pub global activate flutterfire_cli
   ```
4. From the project root, run:
   ```bash
   flutterfire configure
   ```
   This generates a real `lib/firebase_options.dart` for the platforms you
   select, and adds `google-services.json` for Android automatically.

### 2. Install dependencies

```bash
flutter pub get
```

### 3. Run the app

```bash
flutter run
```

## How It Works

- **On launch**, `SplashScreen` plays its intro animation, then hands off to
  `AuthGate`.
- **`AuthGate`** watches `AuthProvider` and shows `LoginScreen` or
  `HomeScreen` automatically — no manual navigation is needed after
  sign-in, sign-up, or sign-out.
- **`TaskProvider`** holds all task state. `HomeScreen` watches it, so any
  change (add, toggle, delete) updates the UI immediately.
- **Tasks are stored locally** via `SharedPreferences`, namespaced by user
  ID (`tasks_<uid>`), so different accounts on the same device don't share
  data. Note that this means tasks do **not** sync across devices — see
  [Extending](#extending) below if you need that.

## Extending

- **Cloud sync**: swap `TaskService` for a Firestore-backed implementation
  (e.g. `users/{uid}/tasks/{taskId}`). `TaskProvider`'s public API
  (`addTask`, `deleteTask`, `toggleTask`, `loadForUser`) wouldn't need to
  change — only `TaskService`'s internals would.
- **Password reset** is already wired up via `AuthProvider.sendPasswordResetEmail`,
  triggered from "Forgot Password?" on the login screen.

## Known Limitations

- No `flutter analyze` / build has been run against a live Flutter SDK in
  this environment — run `flutter pub get && flutter analyze` after
  configuring Firebase to catch any version-specific issues.
- Tasks are local-only (no cross-device sync) until Firestore is added.
